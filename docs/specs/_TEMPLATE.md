## Приоритеты

- **must** — обязательно для релиза. Блокирует приёмку.
- **should** — важно, но допускается отложить с обоснованием.
- **could** — желательно. Реализуется при наличии времени.

## Жизненный цикл требования

```
Черновик → Утверждено → Реализовано → Верифицировано
```

- **Черновик** — требование сформулировано, ожидает ревью.
- **Утверждено** — требование принято в работу.
- **Реализовано** — код написан и покрыт тестами.
- **Верифицировано** — QA подтвердил соответствие критериям приёмки.
